# Agent Orchestration Framework

This is a **meta-project**: a multi-agent orchestration framework that uses Claude Code to coordinate PM and Dev agents for automated software development.

## What this repo IS

The deliverable is the orchestration infrastructure:

- `.agents/` ‚Äî Agent identities (`pm.md`, `dev.md`), orchestrator script, config
- `.claude/commands/` ‚Äî Skills (slash commands) agents use: `gen-spec`, `gen-answer`, `gen-question`, `update-status`

## What this repo is NOT

Everything else is **test data** generated by running the agents. These files exist only to validate the framework and must never be committed:

- `docs/` ‚Äî Example domain documentation (input for PM Agent)
- `pm/specs/` ‚Äî Example specs generated by PM Agent
- `pm/questions/` ‚Äî Example questions/answers from dev‚Üîpm loop
- `pm/tasks/` ‚Äî Example task files created by Dev Agent
- `src/` ‚Äî Example code written by Dev Agent

## Architecture

```
orchestrator.sh
‚îú‚îÄ‚îÄ PM Agent (pm.md)     ‚Äî generates specs, answers questions
‚îÇ   ‚îú‚îÄ‚îÄ /gen-spec        ‚Äî create spec files
‚îÇ   ‚îú‚îÄ‚îÄ /gen-answer      ‚Äî answer dev questions
‚îÇ   ‚îî‚îÄ‚îÄ /gen-question    ‚Äî (used by dev, answered by PM)
‚îî‚îÄ‚îÄ Dev Agent (dev.md)   ‚Äî plans and implements specs
    ‚îú‚îÄ‚îÄ /gen-question    ‚Äî ask PM when blocked
    ‚îî‚îÄ‚îÄ /update-status   ‚Äî transition spec status + sync BACKLOG.md
```

## Key commands

```bash
orchestrator.sh pm-seed <prompt>           # Generate initial backlog
orchestrator.sh pm-answer-pending          # Answer all pending questions
orchestrator.sh dev-implement SPEC-XXX     # Implement a spec (dev‚Üípm‚Üídev loop)
orchestrator.sh dev-implement-next         # Auto-pick and implement next eligible spec
orchestrator.sh dev-auto                   # Unattended: implement all eligible specs continuously
```

Stop `dev-auto` with `Ctrl+C` or `touch .agents/.stop` from another terminal.

## Configuration

`.agents/config.yaml` ‚Äî project-level settings:
- `project.source_dir` ‚Äî where the dev agent writes code (default: `src/`)
- `orchestrator.max_cycles` ‚Äî max dev‚Üípm‚Üídev cycles (default: 3)
- `agents.pm_model` ‚Äî model for PM Agent (default: `opus`)
- `agents.dev_model` ‚Äî model for Dev Agent (default: `opus`)

## Conventions

- Agent identity files use YAML frontmatter + markdown body (same pattern as specs)
- Skills are Claude Code slash commands in `.claude/commands/`
- The orchestrator invokes agents via `claude -p` with scoped `--allowedTools`
- Both agents have web access (WebSearch, WebFetch) for research
- PM Agent has no Bash access (read/write only); Dev Agent has Bash for tests/builds
